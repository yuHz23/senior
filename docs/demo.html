📡 CICIoT 2023 — Features (5 chiều)
flow_duration
Thời gian của một network flow (giây). DDoS floods thường có duration rất ngắn hoặc rất đều đặn bất thường.
Rate
Tốc độ truyền gói tin (packets/sec). Attack traffic thường có rate đột biến cực cao bất thường.
Number
Số lượng packets trong flow. DDoS flood = số packet rất lớn trong thời gian ngắn.
IAT (Inter-Arrival Time)
Thời gian giữa 2 gói tin liên tiếp (giây). Attack: IAT đều đặn gần bằng 0 (scripted flood), không tự nhiên như human traffic.
Tot size (Total Size)
Tổng kích thước payload (bytes). Kết hợp với Rate và IAT tạo thành "temporal fingerprint" của traffic. GRU học toàn bộ pattern này qua 64 timestep.
🌡️ Intel Berkeley — Features (4 chiều)
temperature (°C)
Nhiệt độ môi trường. Bình thường: 18–35°C. SensorFault: >100°C hoặc <-20°C — giá trị vật lý bất khả thi.
humidity (%)
Độ ẩm tương đối. Bình thường: 20–90%. Fault: >100% (cảm biến hỏng báo giá trị ngoài dải vật lý).
light (lux)
Cường độ ánh sáng. Bình thường: 0–2000 lux. Fault: >3000 lux hoặc giá trị âm.
voltage (V)
Điện áp pin cảm biến. Bình thường: 2.0–3.5V. <2.0V = pin gần hết, >3.5V = sự cố điện.
📈 Các chỉ số đánh giá
AUC-ROC
Area Under ROC Curve. Đo khả năng phân biệt Normal vs Attack của model khi thay đổi threshold.
1.0 = hoàn hảo · 0.5 = random · <0.5 = tệ hơn random.
→ Không phụ thuộc vào threshold θ, đánh giá "bản chất" của model.
Accuracy
Độ chính xác tổng thể. = (TP + TN) / (TP + TN + FP + FN)
Tỉ lệ windows được phân loại đúng trong tổng số. Dễ hiểu nhưng dễ bị mislead khi dữ liệu mất cân bằng.
Precision
Độ chính xác của cảnh báo. = TP / (TP + FP)
"Khi model báo ATTACK, bao nhiêu % là đúng?"
Thấp = nhiều False Alarm → người vận hành mất niềm tin vào hệ thống.
Recall
Độ nhạy / Detection Rate. = TP / (TP + FN)
"Trong tất cả attack thực sự, bao nhiêu % được phát hiện?"
Thấp = bỏ sót attack → nguy hiểm nhất cho hệ thống bảo mật. Thesis target: Recall = 1.0.
F1-Score
Harmonic mean của Precision và Recall. = 2 × P × R / (P + R)
Metric cân bằng duy nhất khi quan tâm cả hai. F1 = 1.0 khi P = R = 1.0.
Threshold θ
Ngưỡng phân loại MSE. error > θ → ATTACK.
Được chọn tại percentile 99 của reconstruction error trên validation normal.
→ 1% False Positive Rate. Tăng θ = ít False Alarm nhưng bỏ sót nhiều hơn. Giảm θ = ngược lại.
🔢 Confusion Matrix — 4 trường hợp
✓ True Negative (TN)
Traffic thực sự là Normal và model dự đoán NORMAL.
Tốt — không có False Alarm.
✗ False Positive (FP)
Traffic thực sự là Normal nhưng model báo ATTACK.
False Alarm — gây phiền, giảm niềm tin.
⚠ False Negative (FN)
Traffic thực sự là Attack nhưng model báo NORMAL.
Bỏ sót — nguy hiểm nhất! Hệ thống không phản ứng kịp.
✓ True Positive (TP)
Traffic thực sự là Attack và model dự đoán ATTACK.
Tốt — phát hiện đúng tấn công.
